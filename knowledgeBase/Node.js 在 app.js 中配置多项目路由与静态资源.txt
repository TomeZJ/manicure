Node.js 在 app.js 中配置多项目路由与静态资源
在真实的业务场景中，一个 Node.js 服务往往不只服务**单一项目**，而是需要同时承载多个子系统，例如：
后台管理系统（Admin）
官网 / 门户站点（Web）
H5 页面
历史版本 / 旧系统
API 服务
API 服务升级后的区分
如果每个项目单独起一个服务，不仅**部署成本高**，还会增加 **运维和端口管理复杂度**。
因此，**在一个 Node.js 项目中，通过 app.js 统一配置多项目路由和静态资源**，是非常常见且成熟的工程实践。
一、为什么要在一个 Node.js 项目中承载多个项目？
常见业务场景
一个域名下：
/admin → 后台系统
/home → 官网
/h5 → 活动页面
多个前端项目共用一套 API
API版本划分 → V1、V2
历史项目需要平滑迁移

优点
✅ 统一端口
✅ 统一日志、鉴权、中间件
✅ 降低部署和运维成本
✅ 多项目共享能力（Redis / DB / 监控）

二、推荐的项目目录结构
一个清晰的目录结构，是多项目共存的基础。
project-root
├── app.js
├── package.json
├── router
│ ├── api.js
│ ├── admin.js
│ └── home.js
├── admin
│ ├── index.html
│ └── static
├── home
│ ├── index.html
│ └── static
├── h5
│ └── index.html
└── public
└── common
说明：
admin / home / h5：不同前端项目构建后的产物
router/：API 路由
public/：公共静态资源
三、app.js 的核心职责
在多项目架构下，app.js 主要承担以下职责：
注册全局中间件
配置多项目静态资源
绑定多项目路由入口
统一错误处理
四、基础 app.js 示例（Express）
以下示例以 Express 为例，Koa 思路完全一致。
const express = require('express');
const path = require('path');
const app = express();
五、多项目静态资源配置
1️⃣ 配置 Admin 项目
app.use('/admin', express.static(path.join(**dirname, 'admin')));
app.get('/admin', (req, res) => {
res.sendFile(path.join(**dirname, 'admin/index.html'));
});
访问效果：
http://localhost:3000/admin
2️⃣ 配置 Home 项目
app.use('/home', express.static(path.join(**dirname, 'home')));
app.get('/home', (req, res) => {
res.sendFile(path.join(**dirname, 'home/index.html'));
});
3️⃣ 配置 H5 项目
app.use('/h5', express.static(path.join(**dirname, 'h5')));
app.get('/h5', (req, res) => {
res.sendFile(path.join(**dirname, 'h5/index.html'));
});
4️⃣ 公共静态资源
app.use('/public', express.static(path.join(\_\_dirname, 'public')));

六、为什么需要手动 sendFile index.html？
前端项目（Vue / React）通常是 SPA，存在：
前端路由控制
刷新页面 404 问题
通过显式指定 index.html，可以避免刷新路由失效的问题。
七、多项目 API 路由设计
API 路由拆分
const apiRouter = require('./router/api');
const adminRouter = require('./router/admin');
路由挂载
app.use('/api', apiRouter);
app.use('/admin/api', adminRouter);
示例路径：
/api/user/login
/admin/api/user/list

八、路由与静态资源的优先级
⚠️ 非常重要的一点：
静态资源优先于路由匹配
推荐顺序：
// 静态资源
app.use('/admin', express.static(...));
app.use('/home', express.static(...));
// API 路由
app.use('/api', apiRouter);

九、SPA 刷新 404 的解决方案
问题原因
/home/user/profile
刷新时，Node.js 以为是后端路由。
解决方案：兜底路由
app.get('/home/_', (req, res) => {
res.sendFile(path.join(\_\_dirname, 'home/index.html'));
});
同理适用于 /admin/_。
十、多项目模式下的常见坑
1️⃣ 静态路径冲突❌
app.use(express.static('admin'));
app.use(express.static('home'));
✅ 必须加前缀路径。

2️⃣ API 与前端路由冲突
/admin/api/_ 必须优先匹配
避免 /admin/_ 覆盖 API
3️⃣ 构建路径不一致
前端项目：
publicPath: '/admin/'
必须与 Node.js 中的路由一致。
十一、生产环境建议
1️⃣ Nginx + Node.js 配合
Nginx 托管静态资源
Node.js 只处理 API
但在 中小项目 中，Node.js 直接托管完全可行。
或者直接 Nginx 转发 nodejs 服务。
2️⃣ 项目隔离
路由独立
静态资源独立
互不影响
3️⃣ 日志 & 鉴权统一
多项目共用：
登录态
日志系统
Redis
数据库连接池
十二、总结
在 app.js 中配置多项目路由与静态资源，本质是在解决：
“一个 Node.js 服务如何优雅地承载多个前端项目”
核心要点只有三条：
1️⃣ 路径前缀必须清晰
2️⃣ 静态资源与 API 分离
3️⃣ SPA 需要 index.html 兜底

一句话总结：
路由是骨架，静态是皮肤，app.js 是中枢神经
